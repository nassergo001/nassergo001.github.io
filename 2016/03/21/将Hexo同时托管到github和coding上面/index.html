将Hexo同时托管到github和coding上面

发表于 2016-03-21   |   分类于 Hexo   |  

将本地Hexo博客同时托管到github和coding中，加速浏览。前面我们已经在本地搭建好了Hexo博客
现在可以进入Hexo文件夹，右键选择 git bash进入到git的命令行工具（为什么要进入这个呢？因为这个命令行不光可以执行系统命令，也可以执行git命令，一举两得）

（hexo s 命令就是在本地启动浏览服务，启动成功后你就可以通过localhost:4000在本地测试博客的更新情况，确定无误过后再行操作。）

##配置Github账号

###注册账号
如果已经有了账号，就可以不用注册了。
注册的详细步骤这里就不详细讲解了，按照提示输入用户名、邮箱和密码，然后进入邮箱激活，就可以了，这里的用户名需要记住，后面会用到，关键！！

然后登陆github账号，在右上角新建一个仓库来托管我们的Hexo静态博客

###创建仓库
点击new repository

你将会看到，其他的选项都可以任意填写，但是我用红线框起来的部分必须是如下结构：“你的github用户名” + “.github.io”
例如我的就是NasserGo.github.io(大小写好像无所谓，没有测试)
然后其他的就随便填写。然后点击Creat repository就创建好了。

###设置Github page
创建完成过后就回来到如下界面，点击图中的Setting（红色框起来的）

在新的页面中找到Github Pages，点击红的框中的按钮

在新的页面中什么也不要管，直接拖到最后，点击绿色按钮

然后会进入到一个新的页面，同样的操作，什么都不管，直接点击Publish page

一般情况下不出意外，你就可以通过nassergo.github.io访问你的静态网站了。你也可以返回到仓库的设置页面，找到之前的github page设置的地方，看到如下界面就表示开启成功了。

好了，到现在，github的page服务已经开启了。下面就是将本机Hexo托管到github上去。

##托管Hexo到Github

###生成SSH公钥
什么是SSH公钥，请自行百度，我也是菜鸟，只是知道配置了这个就可以将代码托管到github上面去。本段文章基于不知道或者没有生成过SSH的人所写，知道的可以跳过了。
在桌面右键，打开git bash（什么？ 你右键没有git bash选项？ 那肯定是你没有安装github的windows客户端，我前一篇文章中有讲到），输入下面命令：

email@email.con是你的github邮箱。
后面会提醒你输入key的名称之类的，你可以不用输入，之间3个回车就OK了。
然后在”C盘–用户–你的用户名–.ssh“可以看到两个文件：id_rsa和id_rsa.pub两个文件，其中pub结尾的就是ssh公钥。

生成成功SSH过后，找到本机上的id_rsa.pub文件，打开(我使用的是sublime Text)，复制里面的代码，记得千万不要多复制了一个空格。

###github添加ssh key
回到github中，选择settings（注意这里选择的是账号的设置页面，而不是仓库的设置页面，我们之前开启page服务是在仓库的设置页面开启的。），页面的右上角，点击头像。如下图

然后点击SSH key

在新的页面中点击右上角的new ssh key

然后在按照按下图的提示填写内容，填写好了过后，点击add

到这里，SSH就添加到了github中，添加这个就是一个身份证明，有了这个可以直接向github的某一个库里面提交代码，所以这个不能泄露。
有的人可能要问，我添加了，怎么知道是够添加成功了呢，任意地方右键打开Git Bash输入下面代码：

The authenticity of host ‘github.com (192.30.252.129)’ can’t be established.
RSA key fingerprint is 16:27:xx:xx:xx:xx:xx:4d:eb:df:a6:48.
Are you sure you want to continue connecting (yes/no)? yes #确认你是否继续联系，输入yes
Warning: Permanently added ‘github.com,192.30.252.129’ (RSA) to the list of known hosts.
Enter passphrase for key ‘/c/Users/xxxx_000/.ssh/id_rsa’: #生成ssh kye是密码为空则无此项，若设置有密码则有此项且，输入生成ssh key时设置的密码即可。
Hi xxx! You’ve successfully authenticated, but GitHub does not provide shell access. #出现词句话，说明设置成功。

##配置coding账号
上面我们配置完成了github托管，有的童鞋因为github服务器在国外，访问速度十分缓慢。于是国内的coding代码托管也推出了page服务，同时在微博上公开永久免费使用。

###注册Coding并添加SSH key(公钥)
首先使用你之前注册github的邮箱账号，去coding官网(https://coding.net)注册一个账号。
注册好了过后，先不要创建项目，我们先把SSH添加到coding账户中去，等你注册好的账户，然后选择右边的”账户“–”SSH公钥”，如图

由于是国内的供应商，中文的界面，所以会比较快捷。
还记得我们之前生成的那个ssh文件吗？ 找到他，把他复制到上图中，然后添加。

###新建项目（仓库），并开启page服务
现在新建一个仓库来存放静态博客Hexo（注意，这里有一个关键点，就是仓库的命名和github有一点不同），仓库的命名填写和你coding账户一样！小写就可以了。这里就有没有什么域名后缀了。像我就只需要填写”nassergo“就可以了。
至于公开还是私有，随便你咯。coding的私有库是免费了，github的私有库是收费的，还是国内的好，嘿嘿、

好了，到现在为止，我们的github和coding的pages服务都成功开启了。

到了这里我相信细心的小伙伴就已经发现了github和coding其实就是创建仓库的时候填写项目名字的时候需要注意。

##总结
总结一下这一部分教程：
注册账号–添加SSH–新建仓库(注意命名方式)–开启pages服务–修改Hexo的系统配置文件–托管到github或者coding
如此而已。

##结语
好了，这一部分教程就完了，现在我们成功的将网站分别托管到了github和coding上面。但是我们还只能通过固定而且难记的二级域名去访问，而且很多小伙伴可能对于怎么使用Hexo还是一头雾水。下一篇教程我将针对于Hexo的基本使用、切换模板和绑定域名来讲解。
